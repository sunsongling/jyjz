# 解忧抉择

解忧抉择是一个面向迷茫、纠结、选择困难人群的微信/企业微信小程序。产品主张是“告别纠结，一秒帮你做决定”，通过“天意随机选”和“理智分析选”两种方式，覆盖日常小事和较重要选择。

## 需求文档落地范围

已按《解忧抉择-迷茫人群决策微信小程序需求说明书》完成用户端核心功能：

- 首页：产品 slogan、双模式入口、首次引导弹窗、辅助入口。
- 天意随机选：抽签/转盘双切页；抽签支持 2-10 个指定选项；转盘支持自定义选项与动态等分背景。
- 自定义多选转盘：支持 3-10 个选项、本地保存、独立切页设置选项、转盘抽取、结果记录；转盘背景会根据选项数量动态等分。
- 热门预设题库：干饭选择、出行选择、休闲娱乐、职场小事、穿搭选择；支持分类直接抽签、点选结果、载入转盘。
- 理智分析选：双方案利弊清单、0-10 权重分、灵魂拷问、决策笔记、简易分析报告。
- 历史记录：自动保存随机选择和理性分析记录，支持查看、删除、清空。
- 治愈专区：每日解忧语录、私密树洞、决策小 tips。
- 个人设置：音效开关、振动开关、清除本地缓存、关于说明和反馈入口。
- 结果展示：展示最近一次结果，支持分享入口。
- 隐私策略：免登录、无手机号授权、无头像昵称授权，所有记录本地存储。

后台管理端在需求文档中属于运营系统，包括数据统计、内容管理、反馈管理和系统设置。当前仓库是 uni-app 小程序前端工程，没有后端服务和管理端框架，因此本次交付未实现后台管理端，只在前端内置了本地题库、语录、拷问问题和 tips。后续如需后台，需要新增服务端 API、数据库和管理端项目。

## 技术栈

- uni-app，Vue 2
- 微信/企业微信小程序
- PixiJS 小程序适配，用于自定义转盘页面
- 本地缓存 `uni.setStorageSync` / `uni.getStorageSync`

## 页面结构

```text
pages
├── index/index.vue      # 首页与首次引导
├── random/index.vue     # 天意随机选，二选一快捷抉择
├── game/index.vue       # 自定义多选 Pixi 转盘
├── game-settings/index.vue # 转盘选项设置
├── presets/index.vue    # 热门预设题库
├── rational/index.vue   # 理智分析选
├── history/index.vue    # 历史记录
├── healing/index.vue    # 治愈解忧专区
├── settings/index.vue   # 个人设置
└── result/index.vue     # 结果展示与分享
```

公共数据层：

```text
common/decision-store.js
```

该文件集中维护本地存储 key、默认设置、语录、题库、灵魂拷问、tips、历史记录和树洞笔记读写。

## 本地存储说明

- `jy_intro_seen`：首次引导是否已展示。
- `jy_settings`：音效和振动开关。
- `jy_history`：随机选择和理性分析历史。
- `jy_treehole`：私密树洞内容。
- `decision_card_options`：自定义转盘选项。
- `jy_latest_result`：最近一次结果页数据。

所有数据仅保存在用户本机，不上传服务器。

## 运行方式

1. 使用 HBuilderX 打开项目根目录。
2. 在 `manifest.json` 中配置微信小程序 AppID。企业微信小程序可使用企业微信关联的小程序 AppID。
3. 在 HBuilderX 中选择“运行到小程序模拟器 -> 微信开发者工具”。
4. 在微信开发者工具中预览、真机调试或上传。

当前项目未内置 `@dcloudio` 命令行构建依赖，建议使用 HBuilderX 运行和发行。

## 验收建议

1. 首次进入首页，确认引导弹窗只出现一次。
2. 在“天意随机选”完成一次二选一，确认结果和历史记录生成。
3. 从“热门题库”载入一个分类，进入自定义转盘抽取，确认历史记录生成。
4. 在“理智分析选”填写利弊和笔记，生成报告，确认历史记录生成。
5. 在“治愈专区”保存树洞内容，退出重进确认仍存在。
6. 在“个人设置”清除本地缓存，确认历史和树洞被清空。
